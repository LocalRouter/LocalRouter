# Move catalog.rs out of version control

**Date**: 2026-02-23
**Status**: Completed

## Context

`crates/lr-catalog/catalog/catalog.rs` is a 1.9MB (59k lines) auto-generated file containing 2578 model entries. It's regenerated by `build.rs` from the models.dev API but was committed to git, causing constant churn. Since `build.rs` already regenerates it on every build, there's no reason for it to be tracked.

## Approach

1. **Generated Rust code to `OUT_DIR`**: Standard Cargo pattern. `catalog.rs` is written to Cargo's build output directory instead of the source tree.
2. **CI caching via `actions/cache`**: Persist `catalog/` directory (`.last_fetch` + `modelsdev_raw.json`) across CI runs. The existing 7-day staleness logic in `build.rs` handles refresh.
3. **`modelsdev_raw.json` stays in git** as a seed/fallback for fresh clones with no cache.

## Changes

### 1. `crates/lr-catalog/buildtools/codegen.rs`
- `generate_catalog_code()` accepts `out_dir: &str` parameter
- `CatalogCodeGenerator::new(out_dir)` constructs output path as `{out_dir}/catalog.rs`
- Removed `const OUTPUT_FILE`

### 2. `crates/lr-catalog/build.rs`
- Reads `OUT_DIR` env var, passes to `generate_catalog_code(&out_dir, &models)`
- Updated `rerun-if-changed` to watch `catalog/modelsdev_raw.json` and `buildtools/`
- Kept watching `.last_fetch` too

### 3. `crates/lr-catalog/src/lib.rs`
- Changed `include!` from `CARGO_MANIFEST_DIR` to `OUT_DIR`

### 4. `.gitignore`
- Added `crates/lr-catalog/catalog/catalog.rs`
- Added `crates/lr-catalog/catalog/.last_fetch`

### 5. Git index cleanup
- `crates/lr-catalog/catalog/catalog.rs` — removed from tracking
- `crates/lr-catalog/catalog/.last_fetch` — removed from tracking
- `crates/lr-catalog/catalog/openrouter_raw.json` — deleted (orphaned)

### 6. Dead code removal
- Deleted `src-tauri/src/catalog/mod.rs` — dead duplicate (app uses `pub use lr_catalog as catalog`)
- Deleted `src-tauri/catalog/.last_fetch` — orphaned artifact

### 7. CI caching
- Added `actions/cache@v4` step in `.github/workflows/ci.yml`
- Added `actions/cache@v4` step in `.github/workflows/release.yml` (build-binaries job)
- Key: `catalog-${{ github.run_id }}` (unique per run, always saves)
- Restore key: `catalog-` (always restores most recent)

## What stays in git
- `crates/lr-catalog/catalog/modelsdev_raw.json` (1MB) — seed data for first-ever CI run or cache eviction

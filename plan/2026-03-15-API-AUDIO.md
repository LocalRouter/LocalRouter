# Audio Endpoints (STT + TTS)

## Context

OpenAI, Groq, TogetherAI, and DeepInfra now expose OpenAI-compatible audio endpoints. This adds speech-to-text (transcription/translation) and text-to-speech to LocalRouter, enabling voice-powered applications through the gateway.

## Endpoints

- `POST /v1/audio/transcriptions` — Speech-to-text (Whisper-compatible)
- `POST /v1/audio/translations` — Speech-to-English translation
- `POST /v1/audio/speech` — Text-to-speech

## Provider Coverage (Native OpenAI-Compatible)

| Provider | STT Transcribe | STT Translate | TTS |
|----------|---------------|---------------|-----|
| OpenAI | Y | Y | Y |
| Groq | Y | Y | Y |
| TogetherAI | Y | N | Y |
| DeepInfra | Y | Y | N |
| LocalAI | Y (via Realtime) | N | Y (via Realtime) |

All other providers return 501 Not Implemented.

## Translation Layer Feasibility

| Feature | Translation Feasible? | Complexity | Notes |
|---------|----------------------|------------|-------|
| STT (transcription) | **No** | N/A | Requires specialized audio processing models (Whisper); cannot be emulated via text APIs. Binary audio data must be processed by a speech model. |
| STT (translation) | **No** | N/A | Same as transcription — requires audio model with multilingual capability. |
| TTS | **No** | N/A | Requires specialized voice synthesis models. Cannot be approximated with text completion. |

**Recommendation:** Native support only. No translation layer for audio endpoints.

## Architecture

### New Pattern: Multipart/form-data Input
STT endpoints accept `multipart/form-data` (not JSON), which is new for LocalRouter.
- Add `"multipart"` feature to `axum` in `crates/lr-server/Cargo.toml`
- Add `"multipart"` feature to `reqwest` in `crates/lr-providers/Cargo.toml`
- Use `axum::extract::Multipart` for request extraction
- Provider implementations construct `reqwest::multipart::Form` to forward upstream

### New Pattern: Binary Response
TTS endpoint returns binary audio data (not JSON).
- Return `axum::response::Response` with `Content-Type: audio/mpeg` (or format-specific)
- Provider returns `SpeechResponse { audio_data: Vec<u8>, content_type: String }`

### Body Size Limit
Audio files can be up to 25MB (OpenAI spec). Current global limit is 16MB.
- Apply route-specific 25MB limit using nested Router with its own `RequestBodyLimitLayer`

## Cross-Cutting Features Applicability

| Feature | Applies? | Notes |
|---------|----------|-------|
| **Auth (API Key)** | **Yes** | Same Bearer token auth as other LLM endpoints |
| **Permission checks** | **Yes** | Client mode enforcement + provider access control. Check client has access to the provider hosting the audio model. |
| **Rate limiting** | **Yes** | Per-API-key rate limiting. Estimate based on audio file size (rough: 1 sec audio ~ 25 tokens). For TTS: estimate based on input text length. |
| **Secret scanning** | **Partial** | TTS input text should be scanned. STT audio cannot be scanned (binary). |
| **Guardrails** | **No** | Guardrails scan text content. STT input is binary audio; STT output is transcription text (could scan output, but low value). TTS input is short text (max 4096 chars). Not worth the complexity. |
| **Prompt compression** | **No** | Not applicable — no conversation context to compress. |
| **RouteLLM** | **No** | Audio models are specialized, not interchangeable like chat models. |
| **Model firewall** | **Yes** | Should apply — user should approve which audio models are used. |
| **Token tracking** | **Partial** | OpenAI reports token usage for STT. TTS is character-based pricing, not token-based. Track what providers report. |
| **Cost calculation** | **Yes** | Use pricing from provider. STT: per-minute pricing. TTS: per-character or per-1M-char pricing. |
| **Generation tracking** | **Yes** | Assign generation IDs, track in GenerationTracker. |
| **Metrics/logging** | **Yes** | Record success/failure, latency, cost. |
| **Client activity** | **Yes** | Record activity + emit "audio" event for tray icon. |
| **Free-tier fallback** | **Yes** | Same pattern as chat — if free-tier audio model exhausted, offer paid fallback. |
| **Streaming JSON repair** | **No** | Not applicable. |
